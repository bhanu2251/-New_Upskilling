## Key Design Choices for Engineer Review

1. **IS_DA flag on DIM_ACCOUNT** — derived from account name pattern match on
   'DEPRECIATION'/'AMORTISATION'. Review against actual Chart of Accounts and adjust
   the CASE expression in dim_account.sql if needed.

2. **EBITDA calculation in FACT_INCOME_STATEMENT** — EBITDA = Revenue - COGS - OpEx (excl. D&A).
   D&A is identified via IS_DA=TRUE on DIM_ACCOUNT. Confirm this matches your accountant's
   definition of D&A accounts before sign-off.

3. **NET_DEBT in FACT_BALANCE_SHEET** — populated as NULL. Net Debt (total interest-bearing
   debt minus cash) requires cross-account aggregation. Build a separate analytical view
   on top of FACT_BALANCE_SHEET to compute this at reporting time.

4. **Balance Sheet cumulative logic** — implemented using a self-join with window function.
   On large datasets consider materializing intermediate period_movements as a separate model.

5. **SCD2 strategy=check** — used because Silver tables have no reliable updated_at timestamp
   (all use full-refresh with QUALIFY dedup). The check strategy re-hashes all tracked columns
   on every run — slightly slower but correct for this source pattern.

6. **Intercompany eliminations** — ELIMINATE=FALSE filter applied in FACT_INCOME_STATEMENT
   and FACT_CASH_FLOW. FACT_TRIAL_BALANCE intentionally includes elimination entries
   (IS_ELIMINATION flag) to support reconciliation. Confirm this is the expected behaviour
   with the finance team before delivery.
